# Piano 0607 — Collector sync + thread

> Piano implementato. Riferimento originale: [`.cursor/plans/0607_sync_threads_733418b2.plan.md`](../../.cursor/plans/0607_sync_threads_733418b2.plan.md)

## Sintesi

- **Architettura**: sync + thread, un set di thread per round, overlap completo (nessun gap).
- **Campionamento**: 1 Hz, snapshot **orderbook completo** (bid+ask Up e Down) in memoria ogni secondo.
- **majority_gain**: calcolato **solo a fine round** su ogni riga, lato maggioritario (midpoint), market buy $100 con walk fee-aware sugli asks salvati.
- **HTTP**: solo Gamma + fee_rate a inizio round; zero chiamate durante campionamento.
- **Formato `.bin`**: invariato v2; book snapshots restano in RAM.
- **Log campionamento console**: ogni tick campionato stampato su **stderr** (visibile solo in console, non in `collector.log`) con sec, bid/ask, side, btc, ptb, livelli book — **senza gain**. `collect.bat` redirige solo stdout nel file.
